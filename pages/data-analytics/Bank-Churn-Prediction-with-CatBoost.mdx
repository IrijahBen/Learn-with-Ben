## ⚙️ Step 5: Feature Engineering

Feature engineering is where data science becomes an art! 🎨 We are going to create custom functions to encode our categorical text into numbers, and we'll even use clustering to create brand-new features.

```python
# Define encoding functions
def encoder(df):
    from sklearn.preprocessing import LabelEncoder
    labelencoder = LabelEncoder()
    for col in df.columns:
        if df[col].dtypes=='object':
            df[col]=labelencoder.fit_transform(df[col])
    return df

def Onehot(df):
    from sklearn.preprocessing import OneHotEncoder
    ohe=OneHotEncoder(handle_unknown='ignore',sparse_output=False).set_output(transform='pandas')
    cat_cols=ohe.fit_transform(df.select_dtypes(include=['category']))
    a=df.drop(columns=df.select_dtypes(include=['category']))
    dff=pd.concat([a,cat_cols],axis=1)
    return dff

# Define a K-Means clustering function to create feature bins
def cluster(X):
    from sklearn import cluster
    agglo = cluster.KMeans(n_clusters=5, random_state=0, n_init="auto")
    agglo.fit(X)
    return ((agglo.labels_ + 1) / 5)
```

**Why are these steps necessary?**
1.  **Label Encoding (`encoder`):** Converts simple text categories into integers. 
2.  **One-Hot Encoding (`Onehot`):** Converts categorical features into binary (0 or 1) columns, which prevents the model from assuming that one category is "greater" than another.
3.  **Clustering (`cluster`):** We use unsupervised K-Means clustering to find hidden patterns in features like `Age` and `EstimatedSalary`, grouping similar customers into "bins". This gives the model new perspectives on the data!

### Applying the Transformations

Let's put our functions to work and create some ratios!

```python

X_train_age = train_df[['Age']]
```

## ⚖️ Step 6: Feature Scaling & Train-Validation Split

Features like `Balance` and `Age` have vastly different numerical ranges. We use a `StandardScaler` to normalize them so they play nicely together. 

```python
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import train_test_split

# Standardizing numerical features
ss = StandardScaler()
ss_target = ['CreditScore', 'Age', 'Balance', 'EstimatedSalary']

for i in ss_target:
    x_train[i] = ss.fit_transform(np.asarray(x_train[i]).reshape(-1, 1))
    x_test[i] = ss.fit_transform(np.asarray(x_test[i]).reshape(-1, 1))

# Splitting the data into training and validation sets
x_train, x_val, y_train, y_val = train_test_split(x_train, y_train, test_size=0.33, random_state=42)
```

**What is the code doing?**
We split our original training data into two chunks: 67% for training (`x_train`) and 33% for validation (`x_val`). The validation set acts as a "mock test" so we can evaluate our model's performance on unseen data before making official predictions.

<hr />

## 🧠 Step 7: Model Building and Evaluation

Now for the grand finale! We will train a **LightGBM Classifier**, a fast, high-performance gradient boosting framework that excels at tabular data.

```python
import lightgbm as lgb
from sklearn.metrics import accuracy_score

# Initialize and train the LightGBM model
lgbc = lgb.LGBMClassifier(boosting_type='gbdt', random_state=94, n_estimators=100)
lgbc.fit(x_train, y_train)

# Evaluate on the validation set
y_pred_val = lgbc.predict(x_val)
print("Validation Accuracy:", accuracy_score(y_val, y_pred_val))

# Evaluate on the training set
y_pred_train = lgbc.predict(x_train)
print("Training Accuracy:", accuracy_score(y_train, y_pred_train))

# Generate final predictions for the test set
y_test = lgbc.predict(x_test)
```

### 🏆 Results!
Our LightGBM model achieved an impressive **~86.55% accuracy on the validation set** and **~87.01% accuracy on the training set**! 

Because the training and validation accuracies are so close to each other, we can confidently say our model generalized well and is not severely overfitting.
